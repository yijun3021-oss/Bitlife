# P0 Mobile Life Sim Design

生成日期：2026-06-18

## 已确认决策

- 第一款游戏先开发 P0 内容，目标是做出可玩的最小生命模拟闭环。
- 体验优先级：手机竖屏优先。
- 视觉方向：数据面板感，清晰展示属性、金钱、关系、学校/工作和年度事件。
- 技术栈：React + TypeScript + Vite + PWA。
- 状态管理：Zustand。
- 存档：单档自动保存，使用 localStorage，存档带 version 字段。
- 多语言：P0 从一开始支持多语言，默认简体中文，同时建立英文资源结构。
- 内容策略：参考已下载 Wiki 的系统骨架和触发条件分类，但事件文案、选项、UI 文案、死因描述全部原创。
- 死亡机制：健康驱动死亡。Health、年龄和简单负面状态共同影响死亡风险。

## 产品目标

P0-M 要做出一条完整人生闭环：玩家创建角色，从出生开始逐年推进；每年遇到主要事件并做选择；选择会改变属性、关系、金钱、学校/工作状态和人生日志；角色成年后可以拥有普通工作和收入；当年龄升高或健康长期偏低时，进入死亡风险；死亡后展示结算页。

P0 不做完整疾病系统、资产系统、成就系统、犯罪系统、社交媒体、宠物、王室、Mafia、商城和云存档。这些留给 P1/P2/P3。

## P0 功能范围

### 角色生成

- 支持创建新人生。
- 可输入或随机生成姓名。
- 可选择性别和国家。
- 初始年龄为 0。
- 初始属性包含 happiness、health、smarts、looks。
- 自动生成父母关系，部分人生可生成兄弟姐妹。

### 年龄推进

- 玩家点击“长大一岁”推进一年。
- 每年执行年度结算：
  - 年龄 +1。
  - 根据年龄进入学龄、成年、老年阶段。
  - 成年且有工作时增加收入。
  - 根据年龄、health 和负面状态检查死亡风险。
  - 如果未死亡，生成一个当前年度事件。

### 属性系统

P0 使用四项核心属性：

- happiness：幸福感，受家庭、学校、工作和事件影响。
- health：健康，影响死亡风险。
- smarts：智力，影响学习、工作机会和部分事件结果。
- looks：外貌，P0 主要用于展示和少量事件条件。

属性范围为 0 到 100。所有规则引擎输出必须将属性限制在这个范围。

### 关系系统

P0 只做家庭关系：

- mother
- father
- sibling

每个关系对象包含 id、name、type、closeness、alive。closeness 范围为 0 到 100。P0 关系页展示家庭成员，并支持少量基础互动，例如聊天、陪伴、争吵。

### 学校系统

- 6 岁进入小学阶段。
- 13 岁进入中学阶段。
- 18 岁离开基础学校。
- 学校状态包含 stage、grade、stress。
- 学校事件会影响 smarts、happiness、stress。

P0 不做大学和专业。

### 工作系统

- 18 岁后可找普通工作。
- P0 提供少量普通职业，例如 cashier、office_assistant、cook、driver、support_agent。
- 当前工作包含 jobId、titleKey、salary、years。
- 每年有工作时增加工资收入。
- 工作事件会影响 happiness、health、smarts 或金钱。

P0 不做晋升树、面试细节、特殊职业。

### 活动系统

P0 活动页只做少量基础活动：

- rest：休息，提高 health，花费少量金钱或无花费。
- study：学习，提高 smarts，可能降低 happiness。
- family_time：陪伴家人，提高关系和 happiness。
- doctor_visit：看医生，花钱提高 health 或移除简单负面状态。
- find_job：成年后找工作。

活动和年度事件共用 effect 处理逻辑。

### 年度事件

P0 使用约 65 条原创事件：

- 出生/开局事件：5 条。
- 童年事件：10 条。
- 学校事件：10 条。
- 家庭事件：10 条。
- 成年普通生活事件：10 条。
- 工作事件：10 条。
- 金钱/小事故事件：5 条。
- 死亡/结局事件：5 条。

事件配置只保存 textKey 和 option textKey，不保存直接文案。事件根据年龄、学校状态、工作状态、属性和 tags 筛选。

### 健康驱动死亡

每年推进时计算死亡风险：

- 年龄越高，基础风险越高。
- health < 40 时风险上升。
- health < 20 时风险明显上升。
- simple negative statuses 可增加风险，例如 injured、frail。
- 如果死亡，生成 deathCauseKey 和结算摘要。

P0 不做完整 disease 表，但允许事件添加简单状态。

### 单档自动保存

- localStorage key 使用 `bitliffe.save.v1`。
- 每次状态变化后自动保存。
- 首页检测到存档时显示继续人生。
- 开始新人生会覆盖旧存档。
- 存档结构带 version，便于后续迁移。

### 多语言

P0 支持 zh-CN 和 en-US。

- UI 文案放在 locale 文件。
- 事件文案放在 locale 文件。
- 配置数据只引用 textKey。
- 语言切换写入本地状态。

## 架构设计

应用分三层：UI 层、游戏状态层、规则引擎层。

UI 层负责手机竖屏展示和交互。主要页面包括新人生创建页、人生仪表盘、年度事件卡、关系页、学校/工作页、活动页、死亡结算页。UI 不直接计算规则，只调用状态层动作。

游戏状态层由 Zustand 管理，保存当前单档人生。状态变化后自动写入 localStorage。

规则引擎层是纯 TypeScript 函数，尽量不依赖 React。它负责生成角色、生成家庭、按年龄筛选事件、计算选项结果、推进年龄、处理学校/工作变化、计算死亡风险、生成死亡结算。

## 数据流

1. 玩家点击选择或活动。
2. UI 调用 Zustand action。
3. action 调用规则引擎。
4. 规则引擎返回新状态和日志条目。
5. Zustand 更新状态。
6. 自动保存到 localStorage。
7. UI 重新渲染。

## 成功标准

- 玩家可以在手机竖屏尺寸下创建角色并开始人生。
- 玩家可以逐年推进，直到死亡结算。
- 每年有事件、活动或状态反馈。
- 属性、金钱、关系、学校/工作状态会随选择改变。
- Health 会影响死亡概率。
- 存档刷新后可继续。
- UI 可在 zh-CN 和 en-US 之间切换。
- P0 配置内容为原创文案，不复制 Wiki 表达。
